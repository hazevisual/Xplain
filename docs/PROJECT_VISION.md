# Project Vision

## Context

Complex business and technical processes are hard to understand from plain text. Relationships between steps, components, and dependencies are often lost. `XPlain` solves this through interactive process visualization.

## Product Goal

Build a web application that:

- accepts a process description (text, document, template);
- automatically extracts stages, subprocesses, and components;
- displays the structure as an interactive map;
- supports drill-down levels L1/L2/L3;
- generates clear infographics for teams and stakeholders.

## Target Audience

- Product and Project Managers;
- System Analysts;
- Architects and Engineering Teams;
- Operations Teams and Leadership.

## Product Principles

- Clarity: the same logic must be understandable for both business and engineering.
- Traceability: every visual block must map back to a source in the original input.
- Quality control: each diagram passes automated checks and manual review before publication.
- Versioning: process changes must be comparable and trackable.

## MVP Success Criteria

- Time from upload to first diagram: up to 2 minutes.
- Support at least 3 detail levels (L1/L2/L3).
- Human editing and approval before publish.
- Export to PDF/PNG and sharable link support.
